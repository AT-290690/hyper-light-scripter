About

Hyper Light Scripter is an online editor that loads script left and right lightning fast like they are nothing! No docker containers and no nodue modules. You can do so much with good old html, js and css! The end product created from this editor is an app that can exist for a short period of time - just like snapchat! Why is this of any use? Because that way you can create a ton of apps and share them for free since they exist only when they are are being observed hence cost 0.001% space to maintain.

Command Input:

The command input is the main place where you can run commands. You can also sometimes run such commands from the editor, but the command input will always be able to do so. It will also keep track of history.

Type HELP or COMMON and press enter. This will display a list of all or some commands.

Application:

Pokedex

To view your application click this icon It will open a window on the side where your HTML document is rendered. Upon saving the file - the document will refresh if opened.

Every script requires an index.html file that will be the document shown in the application. If there is no index.html file existing - one will be created when the button is clicked. It will have only a body tag.

File System:

To save a file click the file icon. Or press ctr/cmd + S. To find out if a file is not saved - the number next to the save button will be 1. 0 means that there are no changes needed to be saved. The command SAVE filename.js will also save a file. To create directory - save a file with the directory in the name SAVE /myfolder/file.js

Select files by clicking on their names at the file selection bar. You can also select files with the command DIR which shows all the files in current or specific directory. DIR /myfolder will show all files in folder /myfolder. OPEN /myfolder/file.js will open this file. If there is a subdirectory the command DIR will be prefixed instead. Directories don't have file extentions like files do. To delete a file use DELETE /myfolder/file.js To clear all files in one directory CLEAR /myfolder
To clear the whole project CLEAR
To download a file use the command DOWNLOAD filename.txt
To download the entire project - you need to zip it first ZIP
and then download the package.zip file. You can also upload folders but explore that on your own.

Package Manager:

To deploy script you can use the command:
COMMIT my_awesome_script - prepare the script for deployment Followed by the command:
PUSH - deploy *public script Then you can find the script - it will have your user id suffixed and depending on the type of script (project or snippet or command) it will have either >_ or ... or >>> prefixed. >_my_awesome_script _X_LsbU4ttbZwpBDYSUKhX
If you have registered then you can find your public script by running the command
PUBLIC . 0 - this will list all your public scripts and you can find your private scripts with
PRIVATE . 0 - this will list all your public scripts the dot (.) means match everything and can be replaced with a keyword to search for The number is the page of your scripts (starting from 0) Both can be omited since it defaults to subcommands: . and 0. *To save a script as PRIVATE add the command SECRET as subcommand to PUSH Example:
COMMIT my_awesome_script Followed by the command:
PUSH SECRET Only you will be able to list your private scripts. Lastly - everytime you close your browser - your will package the current script under the name >_last_script_X_LsbU4ttbZwpBDYSUKhX
It would be a private script.
UPLOAD my_script SECRET is a shorthand for the above and by omitting SECRET the script will be uploade as PUBLIC

Inspect:

When code is selected - a chip icon will appear on the right. Clicking it will output the result from the selected code. If there are mutiple outputs (for instance if code is in a loop) - each output will be displayed on a new line If the file is not saved - the clicking the chip button will save it instead of running. Then after the file is saved the chip will appear again and this time it will output the result.

Scripts:

There are 3 types of scripts:

>_sciprt - entire project with multiple files
...snippet - single code snippet (a copy paste) - no files
>>>executable - commands that can be executed with X command instantly
You don't need to type the prefix - it is asumed based on these factors:

if there are files it will be prefixed with >_
if there are no files it will be prefixed with ...
if the command EXE is used it will be prefixed with >>>

If you have files but want to package a snippet then you can use the command FRAG or FRAGMENT it will remove the files from the commit stage and allow publishing only of the editor content at the moment.

Portals:

As mentioned above if you press icon you will see what is your account key currently X_LsbU4ttbZwpBDYSUKhX - this is also your portal. Portals are folders that are created on the server with your unique key as name. When published - your projects will be suffixed by this key. You can share the applications in your portal
URL + YOUR_KEY
https://hyper-light-scripter .herokuapp.com/ data/storage/portals/ X_LsbU4ttbZwpBDYSUKhX
Keep in mind that this link is temporary. It will exist only until you exit Hyper Light Scripter or Hyper Light Scripter goes to sleep (check section sleep for more details). Thus it is useful only for a quick sharing - when you know when the person recieving the link will view it
To send a reliable link you have to publish the script and use the command: LINK >_script_name This will generate a link that looks like this:
https://hyper-light-scripter .herokuapp.com/ data/storage/root/ editor?script=>_script_name
This will create the script in the reciever's portal.
You can load files from other portals with the LOAD portal-key FROM file IN file command.

Builds:

Builds are an easy way to share your creations without the need of the editor. For them you need to have an index.html file in your project. You can create a self building temporal app and share it as link.
It will exist for a brief amount of time (from 10sec to 30 min) After that it has to be accessed by the same link in order to be recreated.

HYPER >_script_name 15
This will generate a link that looks like this:
https://hyper-light-scripter .herokuapp.com/storage/temp/ ?script=orb_5aSck0wal9NLJ-ZNetXFq &lifespan=10&portal=5aSck0wal9NLJ-ZNetXFq
BUILD script_name 15 Will do the same but it will also upload the project and sufix it with _build_ That way it can be filtered in the account menu.

Common commands:

Find:

FIND - find recent scripts
FIND . 2 - find recent scripts from different page (counting from 0)
FIND tree - find a script matching the word tree
FIND tree 3 - find a script matching the word tree from page 3
FIND tree 0 ADD CX find a script matching the word tree from page 3 with suffix and prefix commands
FIND X_LsbU4ttbZwpBDYSUKhX - find scripts for this account
FIND >_ 0 - find scripts that are packages with files
FIND >>> 0 - find commands that can be executed
FIND \... 0 - find code snippets

zZz Sleep:

Hyper Light Scripter is hosted on heroku for free. This means it is a sleeping app. The application will go to "sleep" after 30 minutes of global inactivity. If not a single user interacts with it. If the app is sleeping it will wake up in 3-10 seconds. If you happen to have a session and before the app went to sleep and 30 minutes later you come back you will lose the project you were working on. Expect if before that you cache it. To cache the entire project use the command CACHE ALL. Then after you wake up the app you have to use the command RELOAD. This will recreate your directory and if you have your files cached - it will recreate them as well.
